Three real-world specifications that ship with ARIA.

Authentication — auth.aria

User login, session management, and lockout behavior.

  • Refined SessionToken type (exact length 64)
  • Login contract with on_failure (invalid credentials, account locked)
  • LoginFlow state machine with 5 states and forbidden transitions
  • Rate limiting per IP and per email
aria check examples/auth.aria
aria gen examples/auth.aria -t typescript -o generated/
aria diagram examples/auth.aria -o docs/auth-flow.md

Payment processing — payment.aria

Complete payment + refund + commission split workflow.

  • Money type in cents with min/max refinement
  • 4 contracts: ChargePayment, RefundPayment, CalculateCommission, ProcessPaymentWithCommission
  • steps + compensate (saga pattern)
  • PaymentLifecycle behavior with 8 states, invariants, forbidden rules, named flows
aria check examples/payment.aria
aria gen examples/payment.aria -t typescript -o generated/
aria gen examples/payment.aria -t rust -o generated/

E-commerce order — order.aria

Order lifecycle with cart validation, pricing, fulfillment, and returns.

  • Cross-module references (import Money, Email)
  • Order state machine: draft → confirmed → paid → shipped → delivered
  • Multi-step orchestration via steps + compensate
  • Timeout and retry policies on external service calls
aria check examples/order.aria
aria gen examples/order.aria -t python -o generated/
