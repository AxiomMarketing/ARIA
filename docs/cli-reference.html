ARIA.
Tutorial CLI Language Examples GitHub

CLI Reference

aria <command> [options]
aria --version   # → 0.1.0

aria check

Validate one or more .aria specs.

aria check <file|dir> [--strict] [--draft] [--json]
  • --strict — Fail if contracts lack examples or on_failure
  • --draft — Allow incomplete specs
  • --json — Machine-readable JSON output

aria gen

Generate code from a spec.

aria gen <file> [-t typescript|rust|python|jsonschema] [-o dir] [--json]
  • -t, --target — Target language (default: typescript)
  • -o, --output — Output directory (default: ./generated)

aria diagram

Generate Mermaid state diagrams from behaviors.

aria diagram <file> [-o file.md] [--json]

aria test

Generate vitest test files from examples.

aria test <file> [-o dir] [-f vitest] [--json]

aria implement

AI-implement contract stubs.

aria implement <file> [--ai claude] [-t typescript] [-o dir] [--api-key key] [--model model]
  • --ai — Provider (default: claude)
  • --api-key — Inline API key (or use ANTHROPIC_API_KEY env var)
  • --model — Override model (e.g. claude-sonnet-4-6)
  • Only typescript target currently supported

aria init

Scaffold a new .aria file.

aria init [--module Name] [-o dir]

aria watch

Re-run check (and optionally gen) on file changes.

aria watch <path> [--gen] [-o dir] [-t lang]

aria fmt

Format .aria files.

aria fmt <file|dir> [--check]

aria setup

Add ARIA section to your project's CLAUDE.md.

aria setup [-o CLAUDE.md] [--specs-dir specs] [-t typescript] [--force]

aria-mcp (MCP Server)

Run the ARIA MCP server for Claude Desktop, Cursor, or any MCP client.

npx aria-mcp

Exposes 5 tools: aria_check, aria_gen, aria_diagram, aria_explain, aria_spec.

Configuration:

// .cursor/mcp.json or claude_desktop_config.json
{
  "mcpServers": {
    "aria": { "command": "npx", "args": ["aria-mcp"] }
  }
}

Exit codes

0Success
1Validation, generation, or IO error
