Types

Primitives

Integer, Decimal, String, Boolean, DateTime

Refinement

type Money is Integer
  where self > 0
  where self <= 10_000_000
  unit "cents"

type Email is String
  where self matches /^[^@]+@[^@]+\.[^@]+$/

Records

type Account is Record
  id: String
  balance: Money

Enums

type Status is Enum
  active    --- Operational
  frozen    --- Locked

Lists

type Orders is List of Order

Generics

type Box of T is Record
  value: T

type Result of T, E is Enum
  ok
  err

Computed fields

type Order is Record
  subtotal: Money
  tax: Money
  total: Money computed as subtotal + tax

Contracts

contract Transfer
  --- Doc comment

  inputs
    from: Account
    amount: Money

  requires
    from.balance >= amount

  ensures
    from.balance == old(from.balance) - amount

  on_failure
    when from.balance < amount
      return InsufficientFunds with remaining: from.balance

  examples
    given
      from: { balance: 10000 }
      amount: 3000
    then
      from.balance == 7000

Additional sections

  • effects — sends, writes, creates, reads, deletes
  • depends_on — external service declarations
  • timeout 30 seconds + retry with backoff exponential
  • rate_limit — max 5 per minute per ip
  • steps + compensate — saga orchestration
  • dispatch on method — polymorphic routing to sub-contracts
  • deprecated "reason" — marks contract for removal

Behaviors (state machines)

behavior OrderLifecycle
  states
    draft
    confirmed
    paid

  initial draft

  transitions
    draft -> confirmed
      when items.length > 0

  invariants
    always balance >= 0
    eventually delivered_at exists
    leads_to created -> paid within 24 hours

  forbidden
    confirmed -> draft

  examples
    flow "happy path"
      draft -> confirmed -> paid

Temporal assertions

once X implies YFrom first time X holds, Y must hold
always XX must be true in every state
never XX must never be true
eventually XX must become true at some point
leads_to X -> Y within N hoursTime-bounded liveness
