A contract, end to end

Every .aria contract declares inputs, preconditions, postconditions, failure cases, and examples. The compiler and AI handle the rest.

module Payment
  version "1.0"
  target typescript

type Money is Integer
  where self > 0
  where self <= 1_000_000

type Account is Record
  id: String
  balance: Money
  status: AccountStatus

type AccountStatus is Enum
  active
  frozen

contract TransferFunds
  --- Transfer money between two accounts atomically

  inputs
    from: Account
    to: Account
    amount: Money

  requires
    from.status == active
    to.status == active
    from.balance >= amount

  ensures
    from.balance == old(from.balance) - amount
    to.balance == old(to.balance) + amount
    result.success == true

  on_failure
    when from.balance < amount
      return InsufficientFunds with remaining: from.balance
    when from.status != active
      return AccountFrozen with account: from.id

  examples
    given
      from: { id: "acc_a", balance: 10000, status: active }
      to:   { id: "acc_b", balance: 5000, status: active }
      amount: 3000
    then
      from.balance == 7000
      to.balance == 8000

Why ARIA

Contracts are first-class

requires / ensures / on_failure built into the language. No bolt-on assertions, no hidden invariants.

Examples are mandatory

Every contract has given/then examples. The compiler turns them into automated tests your AI must pass.

ASCII-only, no Unicode tricks

AI tokenizes ASCII best. ARIA uses implies, and, or, exists — no mathematical symbols to mis-decode.

State machines are native

behavior blocks describe lifecycles with states, transitions, invariants, and forbidden. Mermaid diagrams are generated for free.

Multi-target output

One .aria spec compiles to TypeScript + Zod, Rust + serde, Python + Pydantic, and JSON Schema. No ecosystem lock-in.

Incremental specs

Start with a stub, refine as you go. aria check --draft allows incomplete specs; --strict enforces completeness before release.

The pipeline

Specification, verification, generation, implementation, validation — one flow.

1

Write

Author .aria specs. Humans express intent.

2

Check

aria check catches type errors, unknown states, duplicates.

3

Generate

aria gen produces types, schemas, stubs, tests.

4

Implement

aria implement --ai claude fills the stubs via AI.

5

Validate

Auto-generated tests verify the AI respected the contract.

Install

# Install via npm
npm install -g aria-lang

# Or run without installing
npx aria-lang check my-spec.aria

# Check a spec
aria check examples/payment.aria

# Generate TypeScript + Zod
aria gen examples/payment.aria --target typescript -o src/

# Generate a Mermaid state diagram
aria diagram examples/order.aria -o docs/order.md

# Let Claude implement the contracts
ANTHROPIC_API_KEY=sk-... aria implement examples/payment.aria --ai claude -o src/
