## EXECUTION SEQUENCE

### 1. Update CLAUDE.md with Spec-Driven Rules

Read the existing CLAUDE.md. Look for a section `## ARIA Spec-Driven Development`. 

**If it doesn't exist**, append it. **If it exists**, update it with the latest content.

The section to add/update:

```markdown
## ARIA Spec-Driven Development

This project uses ARIA for spec-first development. The `.aria` files in `specs/` are the 
**source of truth**. Generated code in `{output_dir}` is **derived** from the specs.

### Golden Rules

1. **Spec first, code second** — to add a feature, write the `.aria` spec first, then generate.
2. **Never modify generated types** — `*.types.ts` and `*.behaviors.ts` are regenerated by `aria gen`.
3. **Contract bodies only** — in `*.contracts.ts`, only the function body is yours. Signatures come from specs.
4. **Zero `any`** — all types are Zod-validated. If a type is missing, add it to the spec.
5. **Drift = broken** — run `aria drift` before every commit. If it fails, fix the spec or the code.

### File Ownership

| Pattern | Owner | Modifiable? |
|---------|-------|-------------|
| `specs/*.aria` | Human | YES — this is where you define intent |
| `{output_dir}/*.types.ts` | `aria gen` | NO — regenerated from specs |
| `{output_dir}/*.contracts.ts` | `aria implement` + human | BODY ONLY — signatures are generated |
| `{output_dir}/*.behaviors.ts` | `aria gen` | NO — regenerated from specs |
| `{output_dir}/*.test.ts` | `aria gen` + human | YES — add tests, don't remove generated ones |

### Commands

```bash
# Validate specs
npx aria-lang check specs/ --strict

# Regenerate code from specs  
npx aria-lang gen specs/ -o {output_dir}

# AI-implement contract bodies
npx aria-lang implement specs/ --ai claude -o {output_dir}

# Check spec/code alignment
npx aria-lang drift specs/ {output_dir} --fail-on error

# Format specs
npx aria-lang fmt specs/

# Generate diagrams
npx aria-lang diagram specs/ -o docs/diagrams.md
```

### Development Workflow

```
Want to add a feature?
  1. Write/modify specs/*.aria
  2. npx aria-lang check specs/ --strict
  3. npx aria-lang gen specs/ -o {output_dir}
  4. Implement the new contract bodies
  5. npx aria-lang drift specs/ {output_dir} --fail-on error
  6. Run tests

Want to refactor code?
  1. Make your changes
  2. npx aria-lang drift specs/ {output_dir} --fail-on error
  3. If drift detected → update the spec to match OR revert the code
  4. Run tests

Want to understand the domain?
  1. Read specs/*.aria — types, contracts, behaviors
  2. npx aria-lang diagram specs/ -o docs/diagrams.md
  3. Open the Mermaid diagrams
```

if (!processActionEnsures(input, result)) throw new Error("Postcondition violated");

### 2. Generate CI drift check

Check if the project uses GitHub Actions:

```bash
ls .github/workflows/ 2>/dev/null
```

**If `.github/workflows/` exists**, create or update `.github/workflows/aria-drift.yml`:

```yaml
name: ARIA Spec Drift Check
on: [push, pull_request]
jobs:
  aria-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - name: Validate specs
        run: npx aria-lang check specs/ --strict
      - name: Check drift
        run: npx aria-lang drift specs/ {output_dir} --fail-on error
```

**If no CI detected**, add a `package.json` script instead:

```json
{
  "scripts": {
    "aria:check": "aria-lang check specs/ --strict",
    "aria:drift": "aria-lang drift specs/ {output_dir} --fail-on error",
    "aria:gen": "aria-lang gen specs/ -o {output_dir}",
    "precommit:aria": "npm run aria:check && npm run aria:drift"
  }
}
```

**If `auto_mode=false`**, ask before writing CI config:

```yaml
questions:
  - header: "CI setup"
    question: "Add ARIA drift check to your CI pipeline?"
    options:
      - label: "Yes, GitHub Actions (Recommended)"
        description: "Create .github/workflows/aria-drift.yml — blocks PRs if specs drift"
      - label: "Just npm scripts"
        description: "Add aria:check and aria:drift scripts to package.json"
      - label: "Skip"
        description: "I'll configure CI manually"
    multiSelect: false
```

```
